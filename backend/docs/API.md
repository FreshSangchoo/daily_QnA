# Daily Q&A API 초안

## 개요
- 현재 단계에서는 Daily Q&A의 핵심 기능인 질문 조회와 답변 작성/조회/수정/삭제 기능을 우선 구현한다.
- 질문은 미리 준비된 데이터에서 날짜에 맞게 조회하는 방식으로 사용한다.
- 홈 화면에서는 오늘 질문만 보여주기 위해 별도의 오늘 질문 조회 API를 사용한다.
- 날짜별 질문 조회 API는 과거 기록 조회 등 특정 날짜의 질문이 필요할 때 사용한다.

## 공통 규칙
- 날짜 기준은 한국 시간(`Asia/Seoul`)으로 처리한다.
- 질문은 날짜마다 1개씩 제공된다.
- 질문 데이터는 미리 생성되어 있으며, 총 366개의 질문을 사용한다.
- 질문은 조회만 가능하고 수정 및 삭제는 지원하지 않는다.
- 답변은 사용자가 직접 작성한다.
- 답변은 작성 당일에만 수정할 수 있다.
- 삭제된 답변은 즉시 완전 삭제하지 않고, 휴지통 상태로 30일간 보관한다.
- 삭제된 답변은 일반 답변 조회 결과에 포함하지 않는다.
- 휴지통에 있는 답변은 복원할 수 있으며, 복원되면 다시 일반 답변 조회 결과에 포함된다.

## 질문 관련 API

### 1. 오늘의 질문 조회
- 설명: 홈 화면에서 사용하는 API로, 서버가 한국 시간 기준 오늘 날짜를 계산해 질문을 조회한다. 서버의 시간대로 오늘의 날짜를 고정하기 위해 특정 날짜의 질문 조회 API와 분리했다.
- 메서드: `GET`
- 경로: `/questions/today`

#### 응답 예시
```json
{
  "date": "2026-04-06",
  "questionId": 97,
  "content": "오늘 나를 가장 웃게 만든 순간은 무엇인가요?"
}
```

### 2. 특정 날짜의 질문 조회
- 설명: 전달받은 날짜에 해당하는 질문을 조회한다.
- 메서드: `GET`
- 경로: `/questions/{date}`
- 예시: `/questions/2026-04-06`

#### 응답 예시
```json
{
  "date": "2026-04-06",
  "questionId": 97,
  "content": "오늘 나를 가장 웃게 만든 순간은 무엇인가요?"
}
```

## 답변 관련 API

### 1. 답변 생성
- 설명: 특정 날짜의 질문에 대한 답변을 작성한다.
- 메서드: `POST`
- 경로: `/answers`
- 추가 고려 사항: 추후에 이미지, 파일, 영상 등 첨부 파일이 추가될 것을 생각한다.

#### 요청 예시
```json
{
  "date": "2026-04-06",
  "questionId": 97,
  "content": "친구와 점심을 먹다가 했던 대화가 가장 기억에 남았다."
}
```

#### 응답 예시
```json
{
  "answerId": 1,
  "date": "2026-04-06",
  "questionId": 97,
  "content": "친구와 점심을 먹다가 했던 대화가 가장 기억에 남았다.",
  "createdAt": "2026-04-06T14:20:00"
}
```

### 2. 답변 수정
- 설명: 오늘 작성한 답변만 수정할 수 있다.
- 메서드: `PATCH`
- 경로: `/answers/{answerId}`

#### 요청 예시
```json
{
  "content": "오늘은 친구와의 대화가 가장 즐거웠다."
}
```

#### 응답 예시
```json
{
  "answerId": 1,
  "content": "오늘은 친구와의 대화가 가장 즐거웠다.",
  "updatedAt": "2026-04-06T15:10:00"
}
```

### 3. 날짜별 답변 조회
- 설명: 특정 연도, 월, 일 조건으로 답변을 조회한다.
- 메서드: `GET`
- 경로: `/answers`
- 쿼리 파라미터: `year`, `month`, `day`
- 조회 규칙:
  - `year`는 필수값이다.
  - `month`는 선택값이다.
  - `day`는 `year`와 `month`가 모두 있을 때만 사용할 수 있다.
  - 예를 들어 `year=2026&day=6` 형태의 요청은 허용하지 않는다.

#### 요청 예시
- `/answers?year=2026`
- `/answers?year=2026&month=4`
- `/answers?year=2026&month=4&day=6`

#### 응답 예시
```json
[
  {
    "answerId": 1,
    "date": "2026-04-06",
    "questionId": 97,
    "content": "오늘은 친구와의 대화가 가장 즐거웠다."
  }
]
```

### 4. 답변 삭제
- 설명: 답변을 삭제하면 즉시 완전 삭제되지 않고 휴지통으로 이동하며, 30일 동안 보관된다.
- 메서드: `DELETE`
- 경로: `/answers/{answerId}`

#### 응답 예시
```json
{
  "answerId": 1,
  "deleted": true,
  "deletedAt": "2026-04-06T16:00:00",
  "retentionDays": 30
}
```

### 5. 휴지통 답변 조회
- 설명: 삭제되어 휴지통으로 이동한 답변 목록을 조회한다.
- 메서드: `GET`
- 경로: `/answers/trash`

#### 응답 예시
```json
[
  {
    "answerId": 1,
    "date": "2026-04-06",
    "questionId": 97,
    "content": "오늘은 친구와의 대화가 가장 즐거웠다.",
    "deletedAt": "2026-04-06T16:00:00",
    "remainingDays": 30
  }
]
```

### 6. 휴지통 답변 복원
- 설명: 휴지통에 있는 답변을 복원한다.
- 메서드: `PATCH`
- 경로: `/answers/{answerId}/restore`

#### 응답 예시
```json
{
  "answerId": 1,
  "restored": true,
  "restoredAt": "2026-04-07T09:00:00"
}
```
